# README Style Rules

## Required Sections
Every project README must contain the following sections in order:

1. **Title** — `# project-name`
2. **Overview** — One paragraph description of what the project does.
3. **Installation** — `npm install` or equivalent.
4. **Usage** — At least one code example.
5. **Scripts** — Table of available npm scripts.
6. **Contributing** — Brief contribution guide.
7. **License** — License identifier.

## Formatting Rules
- Use ATX-style headings (`#` not underline).
- Code blocks must specify a language identifier.
- Tables must have a header row and separator row.
- No trailing whitespace on any line.
- File must end with a single newline.

## Length Guidelines
- README must be at least 50 lines.
- No section should exceed 80 lines.

## Tone
- Use imperative mood in instructions ("Run the tests", not "You should run").
- Avoid first-person language ("I", "we") unless in a Contributing section.
